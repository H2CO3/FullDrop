Changelog for FullDrop v1.8
    • New, retinized and redesigned icon and splashscreen; thanks for @pon_switch
    • Should work on iOS 4.0, 4.0.1, 4.0.2, 4.1, 4.2 and 4.2.1.
    • Implemented logout for settings.
    • Added file sharing and supported filetypes (if you open a file with "Open with...", you will be prompted the location of the file. It won't be shown immediately, as it would risk the circumvention of the password protection).
    • Added ability to play audio in the background.
    • Remote files now show their size.
    • Added placeholder for password field.
    • Exchanged position of Up and More buttons.
    • Enhanced file viewer: text files now show up in a text view.
    • Fixed bug that prevented uppercase file extensions to be recognoized.
    • Changed default settings so that the app does not initially ask for a password (this is for avoiding confusion).

Changelog for FullDrop v1.5-1
    • Downloads go to the app's own directory: /var/mobile/Library/FullDrop. This is required in order to avoid vulnerabilities connected to permission handling.
Changelog for FullDrop v1.5
    • Downloads go to /var/mobile/Documents from now on to be standard. This is the new default directory also.
    • Wrote some more meaningful informational texts instead of 'loading' all the time.
    • Removed superfulous metadata reloading upon remote file deletion; added reloading data upon file upload.
    • Fixed minor memory leak.
